# リンク基準

> [!NOTE] Permanent Note
> このファイルはpermanentファイル（永久ノート）として、いつでも編集・更新可能です。

> [!IMPORTANT]
> このシステムでは**時系列単調性DAG**を維持するため、**新しいノートから古いノートへの一方向リンクのみ**が可能です。

## リンクの目的

リンクは以下の目的で使用します：

1. **知識の接続**：関連するノート同士を接続し、知識ネットワークを構築
2. **思考の追跡**：思考の変遷を時系列で追跡可能にする
3. **発見の促進**：関連ノートを発見し、新しい洞察を生み出す
4. **DAG構造の形成**：有向非循環グラフ（Directed Acyclic Graph）を自然に形成

## リンクの基本原則

### 1. 時系列単調性（一方向リンク）

**新しいノートから古いノートへのリンクのみが可能**

- ✅ **良い例**：今日作成したノートから、昨日作成したノートへのリンク
- ❌ **悪い例**：昨日作成したノートから、今日作成したノートへのリンク（編集制限により不可能）

**理由**：
- 編集制限ルールにより、過去のノートは編集できない
- 時系列での思考の変遷を正確に記録するため
- DAG構造を維持し、循環参照を防ぐため

### 2. 積極的なリンク作成

新規ノート作成時は、関連する過去ノートへのリンクを積極的に作成してください。

**リンクを作成すべき場合**：
- 過去のノートの内容を参照している
- 過去のノートの内容を発展させている
- 過去のノートの内容と関連している
- 過去のノートの内容を引用している

### 3. ノートの孤立防止

- 最低1つ以上のタグを設定（タグによる接続）
- 可能な限り関連ノートへのリンクを作成（リンクによる接続）
- タグとリンクの両方で、ノートを知識ネットワークに接続する

## リンクの種類と使い方

### 1. 内部リンク（ウィキリンク）

ノート間を接続する基本的なリンク形式。

**基本形式**：
```markdown
[[ノート名]]
```

**表示名の変更**：
```markdown
[[ノート名|表示名]]
```

**例**：
```markdown
量子コンピュータの基本原理については、[[量子ビットの概念]]を参照。

前回の[[量子コンピュータの基本原理_2025-05-04]]の続きとして...
```

### 2. 埋め込みリンク

他のノートの内容を現在のノートに埋め込む。

**基本形式**：
```markdown
![[ノート名]]
```

**特定の見出しを埋め込む**：
```markdown
![[ノート名#見出し名]]
```

**特定のブロックを埋め込む**：
```markdown
![[ノート名#^ブロックID]]
```

**使用例**：
- 前回のノートの続きを書く際に、前回ノートを冒頭に埋め込む
- 重要な定義や概念を他のノートから引用する
- 関連する図表やコードを埋め込む

**例**：
```markdown
![[量子コンピュータの基本原理_2025-05-04]]

上記の続きとして、量子アルゴリズムについて考察する。
```

### 3. ブロック参照

ノート内の特定の段落やブロックを参照する。

**基本形式**：
```markdown
[[ノート名#^ブロックID]]
```

**ブロックIDの取得方法**：
- ノート内の段落にカーソルを置き、`Ctrl + Shift + I`（または右クリック → 「ブロックIDをコピー」）
- ブロックIDは自動的に生成される（例：`^abc123`）

**使用例**：
- 特定の段落を引用する
- 関連する詳細情報を参照する

**例**：
```markdown
[[量子ビットの概念#^abc123]]で説明されているように、量子ビットは重ね合わせ状態をとることができる。
```

### 4. 外部リンク

Webページや外部リソースへのリンク。

**基本形式**：
```markdown
[表示テキスト](URL)
```

**例**：
```markdown
詳細は[公式ドキュメント](https://example.com/docs)を参照。
```

**注意事項**：
- 外部リンクはObsidian Web Clipperで自動的に取得される
- `source`プロパティにURLを記録することを推奨

## リンク付けのワークフロー

### 新規ノート作成時

1. **関連ノートの検索**
   - Floating Search（Alt + F）で関連キーワードを検索
   - Smart Connectionsで意味的類似ノートを確認
   - タグで関連ノートを絞り込み
   - 詳細な判定方法は [[関連ノート判定ガイド]] を参照

2. **リンクの作成**
   - 関連する過去ノートへのリンクを積極的に作成
   - 必要に応じて埋め込みリンクを使用
   - 特定の段落を参照する場合はブロック参照を使用

3. **リンクの確認**
   - ローカルグラフでノート間の接続を確認
   - バックリンクパネルで、リンク先ノートからの接続を確認

### 続きを書く場合（日を跨ぐ）

同じトピックについて日を跨いで書く場合：

1. **新規ノートを作成**
   - タイトルは重複しないように工夫（例：`トピック名_YYYY-MM-DD`）

2. **前回ノートへのリンク**
   - 冒頭で前回ノートへの明示的なリンクや埋め込みを作成
   - 例：`「前回の[[量子コンピュータの基本原理_2025-05-04]]の続きとして...」`

3. **埋め込みの活用**
   - 前回ノート全体を埋め込む：`![[前回ノート名]]`
   - 前回ノートの特定部分を埋め込む：`![[前回ノート名#見出し名]]`

**例**：
```markdown
![[量子コンピュータの基本原理_2025-05-04]]

## 続き：量子アルゴリズムの実装

前回のノートで説明した量子ビットの概念を踏まえ、具体的な量子アルゴリズムについて考察する。
```

## リンクの命名規則

### ノート名の一意性

- ファイル名はユニークである必要がある（Obsidianの仕様）
- 同じトピックを日を跨いで書く場合は、日付を追加：`トピック名_YYYY-MM-DD`

### リンクの解決

- Obsidianは自動的にファイル名の一部一致でリンクを解決する
- 複数の候補がある場合は、完全なファイル名を使用
- ファイル名のリネームは許可されているが、リンクは自動更新される（「内部リンクを毎回更新する」設定がON）

## リンクの管理

### ローカルグラフの活用

- **グラフビュー**：全体の知識ネットワークを俯瞰
- **ローカルグラフ**：現在のノート周辺の接続を確認
- 孤立したノートがないか定期的に確認

### バックリンクの確認

- **バックリンクパネル**：現在のノートにリンクしているノートを確認
- 関連ノートが適切に接続されているか確認

### リンクの整理

- **未解決リンク**：リンク先が存在しない場合、新規ノートを作成するか、リンクを修正
- **孤立ノート**：タグやリンクで接続されていないノートを発見したら、関連ノートへのリンクを追加

## リンクとタグの使い分け

### リンクを使用する場合

- **具体的な関連性**：特定のノートとの明確な関連がある場合
- **参照・引用**：他のノートの内容を参照・引用している場合
- **発展・継続**：過去のノートの内容を発展させている場合

### タグを使用する場合

- **トピック分類**：ノートの主題や分野を示す場合
- **属性・状態**：ノートの状態や属性を示す場合（`#pin`, `#draft`など）
- **検索性の向上**：検索で見つけやすくするため

### 併用の推奨

- **リンクとタグの両方を使用**：ノートを知識ネットワークにしっかりと接続する
- 例：量子コンピュータについてのノート
  - タグ：`#physics`, `#quantum`
  - リンク：`[[量子ビットの概念]]`, `[[量子アルゴリズムの基礎]]`

## 実践例

### 例1：新規ノート作成

```markdown
---
created: 2025-01-15 10:30:00
tags:
  - physics
  - quantum
---

## 量子アルゴリズムの実装

[[量子ビットの概念]]で説明した量子ビットの重ね合わせ状態を利用して、具体的な量子アルゴリズムを実装する方法を考察する。

参考：[[量子コンピュータの基本原理_2025-01-10]]
```

### 例2：続きを書く場合

```markdown
---
created: 2025-01-16 14:20:00
tags:
  - physics
  - quantum
---

![[量子アルゴリズムの実装_2025-01-15]]

## 続き：量子アルゴリズムの最適化

前回のノートで実装した量子アルゴリズムを最適化する方法について考察する。

前回の[[量子アルゴリズムの実装_2025-01-15#^最適化の課題]]で指摘した課題を解決する。
```

### 例3：ブロック参照の使用

```markdown
---
created: 2025-01-17 09:15:00
tags:
  - physics
  - quantum
---

[[量子ビットの概念#^重ね合わせ状態]]で説明されているように、量子ビットは0と1の重ね合わせ状態をとることができる。

この性質を利用して、[[量子アルゴリズムの実装_2025-01-15]]で実装したアルゴリズムを改善できる。
```

## よくある質問

### Q: 過去のノートから新しいノートへのリンクは作成できますか？

A: **できません**。編集制限ルールにより、過去のノートは編集できないため、新しいノートへのリンクを追加することはできません。代わりに、新しいノートから過去のノートへのリンクを作成してください。

### Q: リンクは何個まで作成しても良いですか？

A: 制限はありませんが、**関連性の高いノートにのみリンク**することを推奨します。過度にリンクを追加すると、知識ネットワークが複雑になりすぎる可能性があります。

### Q: 埋め込みリンクと通常のリンクの使い分けは？

A: **埋め込みリンク**は、他のノートの内容を現在のノートに表示したい場合に使用します。**通常のリンク**は、関連ノートへの参照を示す場合に使用します。続きを書く場合など、前回ノートの内容を確認しながら書く場合は埋め込みリンクが便利です。

### Q: リンク先のノートが存在しない場合は？

A: Obsidianでは、リンク先が存在しない場合、リンクをクリックすると新規ノートを作成できます。ただし、意図しないノートの作成を避けるため、リンクを作成する前に、既存のノートを検索することを推奨します。

### Q: ファイル名を変更した場合、リンクは自動更新されますか？

A: はい、「内部リンクを毎回更新する」設定がONの場合、ファイル名を変更すると、そのノートへのリンクは自動的に更新されます。

### Q: 同じノートへのリンクを複数回作成しても良いですか？

A: はい、問題ありません。ただし、同じ段落内で同じノートへのリンクを複数回作成する場合は、読みやすさを考慮して、1回にまとめることを推奨します。

### Q: Smart Connectionsで発見された関連ノートにはリンクを作成すべきですか？

A: Smart ConnectionsはAIによる意味的類似性を発見するツールです。**必ずしもリンクを作成する必要はありません**が、実際に関連性が高いと判断した場合は、リンクを作成することで知識ネットワークを強化できます。
